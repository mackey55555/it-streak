# 問題生成用プロンプト

## AIへのプロンプト例

```
基本情報技術者試験の問題を生成してください。

## 要件
- 4択問題を生成してください
- 各問題に解説を含めてください
- 難易度を1（易しい）〜3（難しい）で設定してください

## カテゴリと問題数
以下のカテゴリごとに指定された問題数を生成してください：

### テクノロジ系（残り270問）
- 数値表現・論理演算: 20問
- ハードウェア（CPU、メモリ、入出力装置など）: 30問
- ソフトウェア（OS、ファイルシステム、ミドルウェアなど）: 40問
- データベース（SQL、正規化、トランザクションなど）: 40問
- ネットワーク（プロトコル、TCP/IP、LAN/WANなど）: 50問
- セキュリティ（暗号化、認証、アクセス制御など）: 30問
- アルゴリズム・データ構造（スタック、キュー、ソート、探索など）: 30問
- プログラミング（オブジェクト指向、設計パターン、テストなど）: 30問

### マネジメント系（残り90問）
- プロジェクトマネジメント（WBS、ガントチャート、クリティカルパスなど）: 30問
- ITIL（インシデント管理、問題管理、変更管理など）: 20問
- アジャイル開発（スクラム、スプリント、XPなど）: 20問
- システム監査: 10問
- 品質管理: 10問

### ストラテジ系（残り90問）
- 経営戦略（SWOT分析、PEST分析、バリューチェーンなど）: 30問
- システム戦略（ERP、CRM、クラウドコンピューティング、DXなど）: 25問
- 法務（個人情報保護法、著作権法、不正アクセス禁止法など）: 20問
- 標準化（ISO、JISなど）: 10問
- 企業活動（コスト管理、ROIなど）: 5問

## 出力形式
以下のJSON形式で出力してください：

```json
[
  {
    "category": "technology",
    "question_text": "問題文",
    "choice_a": "選択肢A",
    "choice_b": "選択肢B",
    "choice_c": "選択肢C",
    "choice_d": "選択肢D",
    "correct_answer": "A",
    "explanation": "解説文",
    "difficulty": 2
  }
]
```

## 注意事項
- 問題文は明確で、誤解を招かない表現にしてください
- 選択肢は全て妥当な選択肢にしてください（明らかに間違っている選択肢は避ける）
- 解説は簡潔で分かりやすくしてください
- 基本情報技術者試験の出題範囲に準拠してください
- 既存の問題と重複しないようにしてください
```

## ファイル形式

生成された問題は、以下のいずれかの形式で保存してください：

### 推奨: JSON形式（`questions.json`）

```json
[
  {
    "category": "technology",
    "question_text": "OSI基本参照モデルにおいて、ネットワーク層の役割はどれか。",
    "choice_a": "伝送路上のビット列の伝送",
    "choice_b": "隣接ノード間のデータ転送",
    "choice_c": "エンドツーエンドのデータ転送の信頼性確保",
    "choice_d": "ネットワーク上の経路選択",
    "correct_answer": "D",
    "explanation": "ネットワーク層（第3層）は、異なるネットワーク間の経路選択（ルーティング）を行う層です。",
    "difficulty": 2
  }
]
```

### 代替: CSV形式（`questions.csv`）

```csv
category,question_text,choice_a,choice_b,choice_c,choice_d,correct_answer,explanation,difficulty
technology,"OSI基本参照モデルにおいて、ネットワーク層の役割はどれか。","伝送路上のビット列の伝送","隣接ノード間のデータ転送","エンドツーエンドのデータ転送の信頼性確保","ネットワーク上の経路選択",D,"ネットワーク層（第3層）は、異なるネットワーク間の経路選択（ルーティング）を行う層です。",2
```

## カテゴリの値

- `technology`: テクノロジ系
- `management`: マネジメント系
- `strategy`: ストラテジ系

## 難易度の値

- `1`: 易しい（基礎的な知識）
- `2`: 普通（標準的な知識）
- `3`: 難しい（応用的な知識）

## 統合方法

生成されたJSONファイルを`seed-questions.ts`に統合するスクリプトを用意します。
